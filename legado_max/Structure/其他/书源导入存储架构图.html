LAYER 2
ViewModel 层 — 导入逻辑处理
ImportBookSourceViewModel.kt
1
格式识别与路由
通过扩展函数判断输入类型：isJsonObject / isJsonArray / isAbsUrl / isUri
2
数据获取
URL类型发起HTTP请求 (okHttpClient.newCallResponseBody)，Uri类型读取InputStream，JSON直接解析
okHttpClient.newCallResponseBody { url(url) }.decompressed().byteStream()
3
GSON 解析
将JSON字符串/InputStream反序列化为 BookSource 对象列表
GSON.fromJsonObject<BookSource>(mText) / GSON.fromJsonArray<BookSource>(inputStream)
4
数据验证
校验 bookSourceUrl 是否为空，无效则抛出 NoStackTraceException
if (it.bookSourceUrl.isEmpty()) throw NoStackTraceException("不是书源")
5
缓存到内存
解析成功的 BookSource 对象存入 allSources 列表，等待用户选择
allSources.add(it) / allSources.addAll(items)
6
对比数据库
遍历 allSources，查询数据库判断是否已存在，标记新增/更新状态
appDb.bookSourceDao.getBookSourcePart(it.bookSourceUrl)
importSelect() → 用户确认后

LAYER 3
辅助层 — 安全校验与数据准备
SourceHelp.kt + AppConfig
用户选择处理
  • 遍历 selectStatus 获取选中书源
  • 应用保留配置: importKeepName/Group/Enable
  • 处理分组: isAddGroup 追加或替换

安全检查
  • is18Plus() 检查网址是否含成人内容
  • 18+ 网址弹出Toast并跳过
  • 仅合法书源传入 DAO insert
fun insertBookSource(vararg bookSources: BookSource) { val bookSourcesGroup = bookSources.groupBy { is18Plus(it.bookSourceUrl) } bookSourcesGroup[true]?.forEach { appCtx.toastOnUi("${it.bookSourceName}是18+网址,禁止导入.") } bookSourcesGroup[false]?.let { appDb.bookSourceDao.insert(*it.toTypedArray()) } Coroutine.async { adjustSortNumber() } }

LAYER 4
实体层 — BookSource Entity & TypeConverters
BookSource.kt
@Entity 注解配置
  • @Entity(tableName = "book_sources")
  • @TypeConverters(BookSource.Converters::class)
  • indices = [Index(value = ["bookSourceUrl"])]
  • @Parcelize — 支持Parcelable传输
TypeConverters (GSON序列化)
  • Converter ExploreRule ↔ JSON String
  • Converter SearchRule ↔ JSON String
  • Converter BookInfoRule ↔ JSON String
  • Converter TocRule ↔ JSON String
  • Converter ContentRule ↔ JSON String
  • Converter ReviewRule → "null" (未实现)
字段名 类型 说明 注解
bookSourceUrl String 主键，书源地址 PrimaryKey
bookSourceName String 书源名称

enabledExplore Boolean 启用发现 @ColumnInfo(defaultValue="1")

ruleExplore ExploreRule? 发现规则 TypeConverter

LAYER 5
DAO 层 — BookSourceDao 数据访问接口
BookSourceDao.kt
写入操作
  • @Insert(OnConflictStrategy.REPLACE)
  • fun insert(vararg bookSource: BookSource)
  • 冲突时自动 REPLACE (根据主键)
更新操作
  • @Update — 更新已有记录
  • @Delete — 删除记录
  • @Query("delete from book_sources where...")
查询操作 (Flow响应式)
  • flowAll() — 流式查询全部
  • flowSearch(key) — 关键词搜索
  • flowEnabled() — 仅启用的
  • flowExplore() — 发现源列表
  • flowGroups() — 分组列表
便捷操作
  • getBookSourcePart(key) — 获取轻量视图
  • has(key) — 是否存在
  • enable(url, enable) — 启用/禁用
  • upOrder() — 更新排序
  • upGroup() — 更新分组
@Dao interface BookSourceDao { // 插入: 主键冲突时替换 @Insert(onConflict = OnConflictStrategy.REPLACE) fun insert(vararg bookSource: BookSource) // 查询: 通过主键获取轻量视图 @Query("select * from book_sources_part where bookSourceUrl = :key") fun getBookSourcePart(key: String): BookSourcePart? // 判断是否存在 @Query("SELECT EXISTS(select 1 from book_sources where bookSourceUrl = :key)") fun has(key: String): Boolean }

轻量视图，仅包含展示字段，用于列表查询优化，避免加载大JSON规则字段：
CREATE VIEW book_sources_part AS select bookSourceUrl, bookSourceName, bookSourceGroup, customOrder, enabled, enabledExplore, hasLoginUrl, lastUpdateTime, respondTime, weight, hasExploreUrl, eventListener, bookSourceType from book_sources
冲突处理策略
当插入的 bookSourceUrl 已存在时：
  • OnConflictStrategy.REPLACE
  • DELETE 旧记录 + INSERT 新记录
  • 保证主键唯一性
  • 新数据完全覆盖旧数据
