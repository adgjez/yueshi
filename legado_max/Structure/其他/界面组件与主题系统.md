# Legado Plus 界面组件与主题系统

## 一、界面顶部标题栏组件

### 1.1 TitleBar 自定义组件

项目中几乎所有界面的顶部标题栏都使用自定义的 **TitleBar** 组件。

**组件位置：** `io.legado.app.ui.widget.TitleBar`

**组件本质：**
- 继承自 `AppBarLayout`（Material Design 组件）
- 内部嵌入 `androidx.appcompat.widget.Toolbar` 作为核心组件
- 通过 XML 布局 `view_title_bar.xml`（亮色）或 `view_title_bar_dark.xml`（暗色）来 inflate Toolbar

### 1.2 TitleBar 的典型用法

在 Fragment 或 Activity 的 XML 布局中使用：

```xml
<!-- 书架页示例 -->
<io.legado.app.ui.widget.TitleBar
    android:id="@+id/title_bar"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:attachToActivity="false"
    app:title="@string/bookshelf" />

<!-- 发现页示例 -->
<io.legado.app.ui.widget.TitleBar
    android:id="@+id/title_bar"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:attachToActivity="false"
    app:contentLayout="@layout/view_search"
    app:layout_constraintTop_toTopOf="parent"
    app:title="@string/discovery" />

<!-- 章节列表页示例 -->
<io.legado.app.ui.widget.TitleBar
    android:id="@+id/title_bar"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    app:contentInsetStartWithNavigation="0dp"
    app:contentLayout="@layout/view_tab_layout" />
```

### 1.3 TitleBar 支持的自定义属性

| 属性 | 类型 | 作用 | 默认值 |
|------|------|------|--------|
| `app:title` | String | 标题文字 | - |
| `app:subtitle` | String | 副标题 | - |
| `app:attachToActivity` | Boolean | 是否自动关联到 Activity 的 ActionBar | `true` |
| `app:fitStatusBar` | Boolean | 是否适配状态栏高度 | `true` |
| `app:fitNavigationBar` | Boolean | 是否适配导航栏高度 | `false` |
| `app:contentLayout` | Reference | 在 Toolbar 内嵌入额外布局 | - |
| `app:themeMode` | Enum | 亮色/暗色主题（0=自动，1=强制暗色） | `0` |
| `app:opaque` | Boolean | 是否不透明背景 | `false` |
| `app:displayHomeAsUp` | Boolean | 是否显示返回按钮 | `true` |
| `app:navigationIcon` | Drawable | 自定义导航图标 | - |
| `app:titleTextAppearance` | Reference | 标题文字样式 | - |
| `app:titleTextColor` | Color | 标题文字颜色 | - |
| `app:subtitleTextAppearance` | Reference | 副标题文字样式 | - |
| `app:subtitleTextColor` | Color | 副标题文字颜色 | - |
| `app:contentInsetLeft` | Dimension | 左侧内容边距 | - |
| `app:contentInsetRight` | Dimension | 右侧内容边距 | - |
| `app:contentInsetStart` | Dimension | 起始内容边距 | - |
| `app:contentInsetEnd` | Dimension | 结束内容边距 | - |
| `app:contentInsetStartWithNavigation` | Dimension | 带导航时的起始边距 | - |
| `app:contentInsetEndWithActions` | Dimension | 带操作按钮时的结束边距 | - |

### 1.4 TitleBar 的代码使用

在 Kotlin 代码中操作 TitleBar：

```kotlin
// 设置标题
titleBar.title = "新标题"

// 设置副标题
titleBar.subtitle = "副标题内容"

// 设置标题文字颜色
titleBar.setTitleTextColor(Color.WHITE)

// 设置图标颜色
titleBar.setColorFilter(Color.WHITE)

// 设置导航按钮点击事件
titleBar.setNavigationOnClickListener { 
    // 返回操作
    activity?.onBackPressed()
}

// 获取菜单并添加项
titleBar.menu.apply {
    add("菜单项").setOnMenuItemClickListener { 
        // 菜单点击处理
        true
    }
}
```

## 三、主题系统架构

项目通过一套完整的主题系统来确保所有界面颜色统一。

### 3.1 主题系统层次结构

```
┌─────────────────────────────────────────────────────────────┐
│                    主题配置层 (ThemeConfig)                   │
│  themeConfig.json → 预置主题配置（名称、主色、强调色、背景色等）    │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  主题存储层 (ThemeStore)                      │
│  SharedPreferences → 持久化当前主题颜色配置                     │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                主题读取层 (MaterialValueHelper)               │
│  Kotlin 扩展属性 → Context.primaryColor, accentColor 等       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────┬──────────────┬──────────────┬─────────────────┐
│  TitleBar    │  Theme控件   │  XML属性引用  │   代码直接获取   │

│  primaryColor│  accentColor │  actionBar   │   扩展属性      │
```

### 3.4 主题颜色获取层 — MaterialValueHelper.kt

**文件位置：** `io.legado.app.lib.theme.MaterialValueHelper`

通过 Kotlin 扩展属性，让 `Context` 和 `Fragment` 都能方便地获取当前主题颜色：

```kotlin
// Context 扩展属性
val Context.primaryColor: Int
    get() = ThemeStore.primaryColor(this)

val Context.primaryColorDark: Int
    get() = ThemeStore.primaryColorDark(this)

val Context.accentColor: Int
    get() = ThemeStore.accentColor(this)

val Context.backgroundColor: Int
    get() = ThemeStore.backgroundColor(this)

val Context.bottomBackground: Int
    get() = ThemeStore.bottomBackground(this)

val Context.primaryTextColor: Int
    get() = getPrimaryTextColor(isDarkTheme)

val Context.secondaryTextColor: Int
    get() = getSecondaryTextColor(isDarkTheme)

val Context.isDarkTheme: Boolean
    get() = ColorUtils.isColorLight(ThemeStore.primaryColor(this))

// Fragment 扩展属性
val Fragment.primaryColor: Int
    get() = ThemeStore.primaryColor(requireContext())

val Fragment.accentColor: Int
    get() = ThemeStore.accentColor(requireContext())

val Fragment.backgroundColor: Int
    get() = ThemeStore.backgroundColor(requireContext())

    get() = requireContext().isDarkTheme
```

**使用示例：**
```kotlin

```

### 3.5 TitleBar 主题应用机制

**文件位置：** `io.legado.app.ui.widget.TitleBar`

在 TitleBar 的 `init` 块中，自动应用主题颜色：

```kotlin
init {
    // ... 其他初始化代码 ...
    
    if (!isInEditMode) {
        // 墨水屏模式
        if (AppConfig.isEInkMode) {
            setBackgroundResource(R.drawable.bg_eink_border_bottom)
        } 
        // 透明导航栏模式
        else if (!opaque && context.transparentNavBar) {
            setBackgroundColor(Color.TRANSPARENT)
        } 
        // 正常模式：使用主题主色
        else {
            setBackgroundColor(context.primaryColor)  // 关键：使用主题主色
            elevation = context.elevation
        }
        
        stateListAnimator = null
    }
}
```

**这就是为什么所有界面的标题栏颜色都与主题保持一致** —— TitleBar 在初始化时自动从 `context.primaryColor` 获取当前主题的主色调并设置为背景色。

### 4.2 自定义控件实现原理

以 `ThemeSeekBar` 为例：

```kotlin
class ThemeSeekBar(context: Context, attrs: AttributeSet) : AppCompatSeekBar(context, attrs) {

    init {
        if (!isInEditMode) {
            applyTint(context.accentColor)  // 自动应用主题的强调色
        }
    }
}
```

以 `ThemeCheckBox` 为例：

```kotlin
class ThemeCheckBox(context: Context, attrs: AttributeSet) : AppCompatCheckBox(context, attrs) {

    private var isUserAction = false

    init {
        if (!isInEditMode) {
            applyTint(context.accentColor)  // 自动应用主题的强调色
        }
    }

    override fun performClick(): Boolean {
        isUserAction = true
        val result = super.performClick()
        isUserAction = false
        return result
    }

    fun setOnUserCheckedChangeListener(listener: ((Boolean) -> Unit)?) {
        if (listener == null) {
            return super.setOnCheckedChangeListener(null)
        }
        super.setOnCheckedChangeListener { _, isChecked ->
            if (isUserAction) {
                listener.invoke(isChecked)
            }
        }
    }
}
```

## 八、主题一致性保障流程

### 8.1 完整流程图

```
用户在设置中选择主题
        ↓
ThemeConfig 加载主题配置
        ↓
ThemeStore 保存到 SharedPreferences
        ↓
MaterialValueHelper 扩展属性读取颜色
        ↓
┌───────────────┬───────────────┬───────────────┬───────────────┐
│   TitleBar    │  Theme 控件   │  XML 属性引用  │   代码直接获取 │
│  setBackgroundColor │  init 中 applyTint │  ?attr/actionBar │  context.primaryColor
│  (context.primaryColor) │  (context.accentColor) │  style          │  扩展属性          │
└───────────────┴───────────────┴───────────────┴───────────────┘
        ↓               ↓               ↓               ↓
   标题栏背景色      控件强调色       系统组件主题     自定义颜色应用
   与主题一致       与主题一致       与主题一致       与主题一致
```

### 8.2 切换主题后的处理

当用户切换主题时，系统会：

1. **保存新主题配置**
   ```kotlin
   ThemeConfig.applyTheme(context, themeConfig)
   ```

2. **更新 SharedPreferences 中的颜色值**
   ```kotlin
   ThemeStore.editTheme(this)
       .primaryColor(newPrimaryColor)
       .accentColor(newAccentColor)
       .backgroundColor(newBackgroundColor)
       .bottomBackground(newBottomBackground)
       .apply()
   ```

3. **重建 Activity**
   ```kotlin
   activity?.recreate()
   ```

4. **所有界面重新创建时从 SharedPreferences 读取最新配置**
   - TitleBar 初始化时读取 `context.primaryColor`
   - Theme 控件初始化时读取 `context.accentColor`
   - XML 中引用的主题属性自动解析为新值

5. **全局颜色统一更新**

## 九、核心要点总结

### 9.1 TitleBar

- **是标题栏的核心组件**，继承自 `AppBarLayout`
- 自动使用 `context.primaryColor` 作为背景色
- 支持丰富的自定义属性（标题、副标题、嵌入布局等）
- 通过 `app:contentLayout` 可在标题栏内添加额外控件

### 9.2 中间内容区

- 根据功能使用 `RecyclerView`、`ViewPager`、`ScrollView` 等标准 Android 组件
- 通常位于 `TitleBar` 下方，通过 `ConstraintLayout` 约束或 `LinearLayout` 垂直排列

### 9.3 主题一致性机制

- **存储层：** `ThemeStore` 使用 SharedPreferences 持久化主题颜色
- **读取层：** `MaterialValueHelper` 提供 Kotlin 扩展属性便捷读取
- **应用层：**
  - `TitleBar` 自动应用 `primaryColor`
  - `Theme` 控件自动应用 `accentColor`
  - XML 通过 `?attr/` 和 `@color/` 引用主题色
  - 代码中直接调用扩展属性获取颜色

### 9.4 自定义 Theme 控件

- 位于 `lib/theme/view/` 目录
- 在 `init` 块中自动调用 `applyTint(context.accentColor)`
- 包括：`ThemeSeekBar`、`ThemeCheckBox`、`ThemeRadioButton`、`ThemeSwitch`、`ThemeProgressBar`、`ThemeEditText` 等

### 9.5 动态染色

- `TintHelper` 提供对各类控件的动态染色功能
- 支持在代码中随时更改控件颜色
- 自动处理不同状态（正常、按下、选中、禁用）的颜色变化
