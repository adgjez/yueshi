## 三、事件发送机制（postEvent）

[EventBusExtensions.kt](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/utils/EventBusExtensions.kt) 中定义了三个发送事件的扩展函数：

```kotlin
// 获取 Observable 对象
inline fun <reified EVENT> eventObservable(tag: String): Observable<EVENT> {
    return LiveEventBus.get(tag, EVENT::class.java)
}

// 立即发送事件
inline fun <reified EVENT> postEvent(tag: String, event: EVENT) {
    LiveEventBus.get<EVENT>(tag).post(event)
}

// 延迟发送事件
inline fun <reified EVENT> postEventDelay(tag: String, event: EVENT, delay: Long) {
    LiveEventBus.get<EVENT>(tag).postDelay(event, delay)
}

// 按顺序发送事件（有序）
inline fun <reified EVENT> postEventOrderly(tag: String, event: EVENT) {
    LiveEventBus.get<EVENT>(tag).postOrderly(event)
}
```

## 四、事件订阅机制（observeEvent）

```kotlin
// AppCompatActivity 订阅事件
inline fun <reified EVENT> AppCompatActivity.observeEvent(
    vararg tags: String,
    noinline observer: (EVENT) -> Unit
) {
    val o = Observer<EVENT> { observer(it) }
    tags.forEach { eventObservable<EVENT>(it).observe(this, o) }
}

// AppCompatActivity 订阅粘性事件
```

## 九、架构总结

整个 EventBus 机制的工作流程如下：

1. **底层引擎**：使用 `LiveEventBus` 库，基于 Android `LiveData` 实现，天然支持生命周期感知。
2. **事件标识**：通过 `EventBus` 常量对象中的字符串常量作为事件 tag。
3. **事件发送**：通过 `postEvent(tag, event)` / `postEventDelay()` / `postEventOrderly()` 三个扩展函数发送。
4. **事件订阅**：通过 `observeEvent(tag)` / `observeEventSticky(tag)` 扩展函数在 Activity/Fragment/Service 中订阅，自动绑定生命周期。
5. **模板方法**：`BaseActivity` 和 `BaseDialogFragment` 在 `onCreate`/`onViewCreated` 中自动调用 `observeLiveBus()`，子类只需覆写该方法即可完成事件订阅。

#### 3️⃣ 订阅事件

订阅通过 `observeEvent` / `observeEventSticky` 扩展函数完成，为三种组件分别提供了扩展：

- `AppCompatActivity.observeEvent()`
- `Fragment.observeEvent()`
- `LifecycleService.observeEvent()`

核心逻辑：

```kotlin
inline fun <reified EVENT> AppCompatActivity.observeEvent(
    vararg tags: String,
    noinline observer: (EVENT) -> Unit
) {
    val o = Observer<EVENT> { observer(it) }
    tags.forEach { eventObservable<EVENT>(it).observe(this, o) }
}
```

关键点：

- 使用 `reified` 泛型让事件类型在编译期确定
- 支持 `vararg tags` 同时订阅多个事件
- `observe(this, o)` 将观察者与组件的 **Lifecycle** 绑定，实现生命周期感知

### 三、关键设计分析

#### 🔹 生命周期感知

LiveEventBus 底层基于 Android `LiveData`，`observe()` 方法会自动绑定 LifecycleOwner。当 Activity/Fragment 处于 `DESTROYED` 状态时，观察者会自动移除，**不会造成内存泄漏**。

配置 `lifecycleObserverAlwaysActive(true)` 意味着即使组件不在前台（非 `RESUMED`），只要没被销毁，也能收到事件。

#### 🔹 粘性事件（Sticky）

`observeEventSticky` 用于订阅**在订阅之前就已发送**的事件。典型场景是音频播放状态：

```kotlin
// AudioPlayActivity 中使用粘性事件，确保进入页面时能拿到最新播放状态
observeEventSticky<Int>(EventBus.AUDIO_STATE) { upPlayState(it) }
```

#### 🔹 EventMessage 通用封装

[EventMessage.kt](file:///g:/Project/legado_Plus/legado_Plus/app/src/main/java/io/legado/app/help/EventMessage.kt) 提供了一种灵活的事件载体，支持通过 `what`（Int）或 `tag`（String）标识，携带 `obj`（Any）数据：

```kotlin
EventMessage.obtain("myTag", myData)  // 创建带 tag 的事件
EventMessage.obtain(1001, myData)      // 创建带 what 的事件
```

但实际项目中大部分场景直接用具体类型（`String`、`Int`、`Bundle`、`ArrayList`）作为事件数据，EventMessage 使用较少。
